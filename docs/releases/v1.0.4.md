# v1.0.4 Release Notes

Release date: 2026-06-02

## Summary

v1.0.4 is a backward-compatible patch release focused on Office rendering fidelity,
chart correctness, and clearer optional PDF.js fallback setup for SmartArt/EMF preview
images.

The public API remains compatible. Ordinary PPTX rendering does not require code
changes. Consumers that rely on EMF files with embedded PDF previews should install
`pdfjs-dist` and pass explicit `pdfjs.moduleUrl` and `pdfjs.workerUrl` values.

## Added

- Configurable `pdfjs` fallback options for `PptxViewer`, legacy `PptxRenderer`, and
  headless `renderSlide()`.
- Exported `PdfjsOptions` and `PdfjsConfig` types.
- Regression coverage for chart grouping, chart labels, EMF-PDF async rendering, text
  fidelity, picture shape properties, and slide readiness.
- Repository `AGENTS.md` guidance for future agent-assisted maintenance.

## Changed

- PDF.js fallback rendering is optional and no longer bundled into the core library
  output by default.
- Local test pages configure PDF.js explicitly for EMF-PDF preview rendering.
- Documentation now explains EMF fallback boundaries and clarifies that full EMF/WMF
  vector rendering is not supported.
- Chart, text, image, table, stroke, and theme-font rendering now follow Office
  behavior more closely across real-world decks.

## Fixed

- EMF-embedded PDF fallback icons now render through isolated PDF.js worker URLs.
- `SlideHandle.ready` can be awaited before screenshots or exports that depend on async
  fallback previews.
- Combo, line, scatter, stacked, 100% stacked, pie, doughnut, radar, and chart label
  rendering mismatches were corrected.
- Picture shape fills, outlines, shadows, and related properties are now preserved.
- Text sizing, wrapping, vertical labels, bullet colors, hyperlink colors, and inherited
  theme fonts were corrected in several Office-generated decks.
- Explicit shape lines without a width now preserve Office's visible default stroke.

## Migration Notes

No migration is required for ordinary PPTX rendering.

To render EMF files that contain embedded PDF previews, configure PDF.js:

```ts
const pdfjs = {
  moduleUrl: new URL('pdfjs-dist/build/pdf.min.mjs', import.meta.url).toString(),
  workerUrl: new URL('pdfjs-dist/build/pdf.worker.min.mjs', import.meta.url).toString(),
};

await PptxViewer.open(buffer, container, { pdfjs });
```

Set `pdfjs: false` to disable EMF-PDF fallback rendering.
